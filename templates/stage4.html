Introduction To Networks

Web of Networks

In Information technology, a netwok is generally defined as a series of points or nodes interconnected by communication paths. These can than be extended to other networks and may also contain subnetworks, thus aiding to communication and exchange of information without actual, device to device connection (no direct channel). However, there is a lot of versatality to networks than just a series of nodes and routers. Internet is one such 'type' of Network, widely known as 'Computer Network' specifically though, Internet is more of a concept than an actual tangible entity, and it relies on a physical infrastructure that connects one network to other networks.


Latency & Bandwidth

We can reliably measure network in two ways : 1. Latency 2. Bandwidth.
Latency is the time that it takes for a message to reach from its source to the destination where it was requested. Bandwidth defined as the amount of informaation that can be transdered per unit time. Its unit is mbps - MEGA Bits Per Second. Refer this. Both concepts seem inversely relational, just like time and frequency. Messages are transferred bitwise. A series of 1's and 0's bundled together which when broken depicts a character. This is also called encoding.

Network Organization and transmission

Computer networks or Internet network differs in the transmission media used to carry their signals or messages, the communications protocols used to organize network traffic, the network's size i.e. the number of bytes in a data-segment( MTU ), network topology and organizational intent. In most cases, communications protocols are layered on (i.e. work using) other more specific or more general communications protocols, except for the physical layer that directly deals with the transmission media(wired or wireless). Web Server

How a web server works and why are they important? At the most basic level, when we type in an address i.e. (Uniform Resource Locator) say "http://www.example.com/foo?p=1/sample.gif", and we see a page on our Computer screen. Behind the scenes ., the browser actually breaks the URL into :
  1. PROTOCOL - Hyper Text Transfer Protocol - HTTP
  2. Server Name - Domain Name (Host)
  3. File Name (Path along with file name)
The Server name (www.example.com) translates to an IP ADDRESS which our browser connects with, and directs through path (foo?p=1/) to get the file name (sample.gif). HTTP is the protocol or language used to make a request in the form of a Query Parameter which mostly a GET Request asking for the given file name. The Server machine makes its service available through numbered path, usually visible at the end pf the URL called port, eg. http://www.xxxx.yyyy.com:918. When no port number is specifiec we assume its the defult port: 80. AT times, ther are extra FRAGMENTS usually seperated by teh "#" sign eg. http://example.com:80/toys?p=foo#blah. Fragments are used to particularly refernce a Page we are looking at. In the URL, it comes after query parameter. Hereis an explanation on how requests are sent by HTTP. The HTTP response is followed by response headers. There are two main classifications of the types of responses a server gives : 1. Static 2.Dynamic. Here is the basic difference. Mostly all websites now a days are dynamic which are made using WEB APPLICATIONS.

String Substitution & Advanced String Substitution

String Substitution is used to sustitute a variable form data which has to be reused and generated every time while coding. This is specifically used for Validating the variable data and resuing the same else where if required, before storing it onto server database. This is in interesting concept to be used. For eg. 
                        "some bold text"
If we had to use the above string everywhere everytime, it would be difficult task. Instead we can use string substitution to reuse the 
                        "%s" % VARIABLE
Now we can sustitute the variable to which ever value we need. Similary if its multiple variable sustitution can be done as below 
                        "some text %s then some text %s" %(variable1, variable2)

What happens if you want to include the same variable twice in the same string? We can do this by including a name identifier in parentheses: 

                        "text %(NAME)s text" % {"NAME": value}
Now, instead of just having the variables at the end of the string, we can include a dictionary that maps NAME to a value. NAME can appear in the string multiple times, and we can have variable names which changes accordingly everywhere. In our prjetcs we have wpplied teh above string substitution on multiple string which is an HTML form We can summerize this as below: 1. Validate user data (use string substitution) & escape if requeired 2. Provide valid error mssage blocks 3. Use redirection on successful receipt on valid data.

Databases

Databases is how we store all the information we have been collecting from users. It can be defined as a program that stres an retrieves lareg amount of structured data. Databses can refer to a program or a machine or a group of machines used to store your applications. Servers once prompted for data draws it from DB and sends it back to the user. This process of querying DB for data has to be done in a structured manner giving rise to a language called Stucrured Query Language (SQL) which is the most common DB used.
There are a lot of Data bases, a few are open source, in the market. But the format in which this Data is stored in remains the same. They involve Tables with a list of Horizontal rows and Vertial Columns. Every HR will show a new entry of data as defined in VC. At times there might be expansion of Column into another table of rows. Manually querying data throught this can be Error prone, tedious and slow due to the custom codes we end up generating. Hence teh use of sturctured language. Clickhere to understand the types of DB. SQL was invented in 1970's. SQL existed long before the WEB. A Query example is:

 SELECT * FROM links WHERE id = 5;
